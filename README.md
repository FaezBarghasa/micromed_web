> **REV_3.12** // SECURE HARDWARE CONTRACTING CELL & DIAGNOSTIC INTERACTION INTERFACE

**MICROMED** is a high-fidelity, retro-futuristic Single Page Application (SPA) designed to simulate the diagnostic interface and contracting portal of a fictional mission-critical embedded systems hardware firm. 

The application utilizes a rich, premium visual design reminiscent of industrial CRT displays, custom printed circuit board (PCB) substrates, and tactical command terminals.

---

## 🛠 Tech Stack

- **Core Structure**: HTML5 (Semantic elements)
- **Styling**: Tailwind CSS (structural layout utility) & Vanilla CSS (custom design tokens, CRT scanlines, and glow aesthetics)
- **Programming Language**: TypeScript (Strict compile configuration)
- **Development & Build Tooling**: Vite & TypeScript Compiler (`tsc`)

---

## 📡 Core Interactive Features

1. **Interactive 3D Hologram Explorer**
   - A custom, canvas-based 3D vector graphics rendering engine built from scratch.
   - Projecting 3D vertices into 2D screen coordinates using rotation matrices and orthographic transformations.
   - Interactive orbit controls allowing mouse-drag rotation with physics-based momentum damping.
   - Render models for industrial components: `STM32 (Blue Pill)`, `ESP32 Module`, `TRIAC Switch`, `Power Transistor`, and `Galvanic Diode`.

2. **Procedural PCB Substrate Background**
   - An animated, full-screen canvas that dynamically generates a multi-layer PCB substrate.
   - Draws copper traces, solder pads, vias, and runs interactive high-speed electron animations along the signal lines to represent active data buses.

3. **Onboard Web Audio Synth Engine**
   - An integrated audio synthesizer powered by the browser's Web Audio API.
   - Generates authentic 8-bit hardware beep tones and pitch sweeps upon user interaction (such as system routing changes, menu selections, and telemetry submissions).

4. **Multi-Language Jumper Protocol (EN / FA)**
   - Fully localized in English and Persian (Farsi).
   - Features dynamic bidirectional layout adjustment (`dir="ltr"` / `dir="rtl"`) with specialized font mapping (`JetBrains Mono` / `Share Tech Mono` for English, `Vazirmatn` for Persian).

5. **Secure Telemetry Uplink (Contracting Form)**
   - Simulates a terminal-style data uplink to request custom board specifications.
   - Provides animated output logs documenting simulated cryptographic handshakes, packet generation, and data bus routing registers.

6. **Metastate Blueprint Interrogator (Team Blueprint)**
   - Interactive organization grid allowing operations managers to review functional credentials and operational focus areas for team nodes.

## 📂 Project Architecture

The application code is highly modularized, with isolated components managing dedicated sections of the application state and behavior.

```
micromed/
├── dist/                  # Production build output (generated by Vite)
├── src/                   # TypeScript Source Code
│   ├── audio.ts           # Web Audio API synthesizer for hardware beeps
│   ├── datasheet.ts       # Dynamic product spec querying and sheet loading
│   ├── form.ts            # Secure telemetry uplink submission and logging
│   ├── hologram-models.ts # Static 3D coordinate geometry for component models
│   ├── hologram.ts        # 3D canvas rendering engine, math projections, and orbit drag
│   ├── i18n.ts            # Translation dictionary, localization jumper, and RTL controller
│   ├── main.ts            # Core entry point, window binders, and event orchestrator
│   ├── pcb.ts             # Procedural PCB track & electron canvas animator
│   ├── router.ts          # SPA view transition and navigation register
│   ├── startup.ts         # Bootloader terminal sequence animation logs
│   ├── state.ts           # Global application state container and setter actions
│   ├── team.ts            # Interactive Metastate team node selection handler
│   └── types.ts           # Common interface contracts and window declarations
├── index.html             # Main HTML layout, head imports, and structure
├── package.json           # Scripts, dependencies, and Vite config
├── resources.json         # Static text resources, translations, and component datasheets
├── style.css              # Custom styling, CRT overlay, scanlines, and neon themes
└── tsconfig.json          # TypeScript compiler configuration options
```

## 🚀 Getting Started

Ensure you have [Node.js](https://nodejs.org/) installed on your machine.

### 1. Installation
Clone the repository, navigate to the project directory, and install development dependencies:
```bash
npm install
```

### 2. Launch Development Server
Start Vite's fast local development server:
```bash
npm run dev
```
Open the output URI (typically `http://localhost:5173`) in your web browser.

### 3. Production Build
Compile TypeScript and bundle assets for production deployment:
```bash
npm run build
```
The compiled, production-ready static assets will be located in the `dist/` directory.

### 4. Preview Production Build
Locally run the built production files:
```bash
npm run preview
```

## 🎨 Design System

The application layout uses strict design tokens defined via CSS custom variables in [style.css](file:///home/jrad/WebstormProjects/micromed/style.css):

- **Substrate Tone**: `--color-pcb-dark` (`#0a0c0d`) - A matte, carbonaceous black.
- **Trace Accent**: `--color-silkscreen` (`#00ff66`) - A vibrant neon green.
- **Trace Dim**: `--color-trace-dim` (`#003314`) - Dark copper forest green for underlying circuits.
- **Glow & Glassmorphism**: Neon box shadows and low-opacity backing backdrops (`backdrop-blur-sm`).
- **CRT Scanlines**: Implemented using a recurring CSS gradient overlay on the body tag to replicate cathode ray tube monitor displays.
